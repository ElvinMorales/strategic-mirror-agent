# Session Loop

1. Load direction, Memory, State, Knowledge, connector registry, and Guardrails.
2. Ingest new input from the user or approved connectors.
3. Classify the input as durable Memory, transient State, reference Knowledge, output-only material, or ignored material.
4. Reconcile against the stored model.
5. Act on the request.
6. Validate voice, confidentiality, connector safety, claims, and no psychoanalysis.
7. Write back only to approved layers.
8. Flag time-sensitive items, confidentiality risks, boundary concerns, approval requirements, and stale connector data.
