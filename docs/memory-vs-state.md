# Memory vs State

Memory and State are separate because they answer different questions.

Memory asks: would this still be useful in three months if nothing was actively worked on?

State asks: is this part of a live situation, decision, deadline, or workflow?

## Memory

Memory may include stable communication preferences, career goals, recurring strengths, recurring traps, durable values, trusted collaborators, and long-running career narrative.

Memory should not include current task status, temporary deadlines, raw logs, full message threads, one-off emotional spikes, or private material that should not be retained.

## State

State may include active projects, pending replies, open decisions, deadlines, recent developments, next actions, uncertainty, and stale-after dates.

State should not include stable identity facts, durable goals, reusable lessons, or private logs that should not be retained.

## Classification Test

If the item will go stale soon, put it in State. If it remains a durable fact after the live situation ends, consider Memory. If it is reusable but not personal context, put it in Knowledge.
